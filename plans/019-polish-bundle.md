## 3. Insert-Mode Line-Editing Bindings Audit

**Problem:**  
Insert mode lacks Ctrl-U (kill line), Ctrl-A (beginning), Ctrl-E (end) bindings. Decision needed: add them to Keymap.zig or declare them buffer-side only?

**Evidence:**  
- Keymap.zig:196–197 defines insert mode with only Esc → normal.
- ConversationBuffer.zig:489–516 already handles Ctrl-W (deleteWordFromDraft).
- Comment at ConversationBuffer.zig:483–488 states "orchestrator strips keybindings before this runs" and handleKey processes "insert-mode editing of the draft buffer."

**Proposed Change:**  
**Decision: Document the existing design; do not add keybindings.**

Rationale: ConversationBuffer already handles Ctrl-W internally (line:493–496). The architecture delegates line editing to the buffer layer, not the keymap. Add a comment to Keymap.zig (near the insert-mode section) clarifying this design choice:

```
// Insert-mode line editing (Ctrl-W, Ctrl-U, etc.) is handled by 
// ConversationBuffer.handleKey(), not the keymap. See 
// ConversationBuffer.zig:483–516.
```

**Steps:**  
- Add the clarifying comment to Keymap.zig (around line 196).
- No new keybindings or buffer methods needed.

**Verification:**  
- Comment is present and clear.
- Ctrl-W still works in insert mode (already tested).
- No new test cases needed; existing tests pass.
