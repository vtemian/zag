# Multi-Session + Persistence Design

**Date:** 2026-04-16

Sessions are like files. Each buffer is a session. JSONL storage with metadata. No tabs. Splits arrange sessions on screen. Session browser is a future plugin.

## Session Model

A session is one conversation thread with one agent. It has:
- Unique ID (UUID)
- Human-readable name (auto-generated by LLM after first exchange)
- A JSONL file (append-only conversation log)
- A meta.json file (summary info for listing/browsing)
- A working directory
- A model identifier

## Storage

```
.zag/sessions/
  a1b2c3d4.jsonl
  a1b2c3d4.meta.json
  f9e8d7c6.jsonl
  f9e8d7c6.meta.json
```

Per-project directory (`.zag/` in cwd).

### meta.json

```json
{
  "id": "a1b2c3d4",
  "name": "fix auth middleware bug",
  "created": "2026-04-16T01:00:00Z",
  "updated": "2026-04-16T01:15:00Z",
  "model": "anthropic:claude-sonnet-4",
  "cwd": "/Users/vlad/projects/ai/zag",
  "message_count": 12
}
```

Updated on every append to the JSONL.

### JSONL entries

Each line is one JSON object with a `type` field:

```jsonl
{"type":"session_start","id":"a1b2c3d4","name":"","model":"anthropic:claude-sonnet-4","ts":"2026-04-16T01:00:00Z"}
{"type":"user_message","parent_id":null,"content":"fix the auth bug","ts":"..."}
{"type":"assistant_text","parent_id":"u001","content":"I'll look at the auth code...","ts":"..."}
{"type":"tool_call","parent_id":"a001","name":"read","input":"{\"path\":\"src/auth.zig\"}","ts":"..."}
{"type":"tool_result","parent_id":"t001","content":"...file contents...","is_error":false,"ts":"..."}
{"type":"info","parent_id":null,"content":"tokens: 895 in, 85 out","ts":"..."}
{"type":"session_rename","name":"fix auth middleware bug","ts":"..."}
```

Entry types map to Buffer.NodeType:
- `user_message` -> .user_message
- `assistant_text` -> .assistant_text
- `tool_call` -> .tool_call
- `tool_result` -> .tool_result
- `info` -> .status
- `error` -> .err
- `session_start` -> metadata only, not a node
- `session_rename` -> updates meta.json name

## Startup

- `zag`: new empty session
- `zag --session <id>`: resume specific session
- `zag --last`: resume most recently updated session

## Auto-save

Entries are appended to JSONL as they happen:
1. User presses Enter: append `user_message` entry
2. Agent text delta complete: append `assistant_text` entry
3. Tool call decided: append `tool_call` entry
4. Tool result received: append `tool_result` entry
5. Token info: append `info` entry

No explicit save command. The file is always up to date.

## Auto-summary (session naming)

After the first assistant response, send a separate (cheap) LLM call:

```
System: Summarize this conversation in 3-5 words. Return only the summary, nothing else.
User: <first user message>
Assistant: <first assistant response>
```

Use the same provider. Store the result as `session_rename` entry and update meta.json.

## Loading a session

1. Read the JSONL file line by line
2. For each entry, create the corresponding Buffer node
3. Rebuild the message history (for continuing the conversation with the LLM)
4. Set the buffer scroll to bottom

The message history (types.Message list sent to the LLM) is reconstructed from the JSONL entries. `user_message` entries become user Messages. `assistant_text` + `tool_call` entries become assistant Messages. `tool_result` entries become tool result Messages.

## Splits and sessions

When you split (`Ctrl+W v`), the new buffer is a new session (new JSONL file). Each buffer has a `session_id` linking to its JSONL file.

When you close a split, the session remains on disk. You can reopen it later.

## Layout changes

Remove tabs from Layout. Simplify to splits only. Each leaf holds a Buffer which is a session.

## SessionManager API

```zig
pub const SessionManager = struct {
    sessions_dir: []const u8,
    allocator: Allocator,

    pub fn init(cwd: []const u8, allocator: Allocator) !SessionManager;
    pub fn createSession(model: []const u8) !Session;
    pub fn loadSession(id: []const u8) !Session;
    pub fn listSessions() ![]SessionMeta;
    pub fn deleteSession(id: []const u8) !void;
};

pub const Session = struct {
    id: []const u8,
    meta: SessionMeta,
    file: std.fs.File,       // open JSONL file for appending

    pub fn appendEntry(entry: SessionEntry) !void;
    pub fn close() void;
};

pub const SessionMeta = struct {
    id: []const u8,
    name: []const u8,
    created: []const u8,
    updated: []const u8,
    model: []const u8,
    message_count: u32,
};

pub const SessionEntry = struct {
    type: EntryType,
    content: []const u8,
    // ... other fields per type
};
```
